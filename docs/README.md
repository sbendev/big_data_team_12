# TheatreIQ — Movie Recommender & Scheduling System

TheatreIQ is a two-stage machine learning recommender that generates **Movie Match Scores** and **weekly screening schedules** for independent movie theaters. Given a theater's historical programming and demographic audience profile, it ranks films by predicted audience fit and greedily assigns them to screens and time slots.

---

## How It Works

```
Film Times.csv ──► TMDB Enrich ──► Synthetic Performance ──┐
MovieLens 100K ──► ALS ──► Segment Vectors ────────────────┤
                                                            ▼
                                              XGBoost Match Score Grid
                                                            ▼
                                     Greedy Schedule + Missed Opportunity Analysis
```

**Stage 1 — ALS Collaborative Filtering**  
Trains on MovieLens 100K explicit ratings to learn latent taste vectors for 8 demographic segments (age × gender bins). Films not in MovieLens are "bridged" via title match or genre cosine similarity.

**Stage 2 — XGBoost Ranker**  
Scores every (segment × film × showtime slot) triple using 74 features: segment ALS vectors, item ALS vectors, 19 genre flags, 6 TMDB metadata features, and 8 slot/calendar features. Output is a Match Score from 0–100.

**Scheduler**  
Greedy assignment fills each (screen × slot) cell with the highest-scoring available film, with optional archetype boosts for theater type. Films are labeled `high_confidence` (≥75), `promotional_support` (50–74), or `consider_dropping` (<50).

### Film Times (`Film Times.csv`)
Historical and synthetic showtime data scraped from Twin Cities theaters.

Date range: Feb 2021 – May 2026 (includes synthetic future programming).  
Columns: `Date`, `Day`, `Showtime`, `Film Title`, `Film Year`, `Venue`

### MovieLens 100K (`ml-100k/`)
100,000 explicit ratings used for ALS training. Place the `ml-100k/` directory in the project root.

### TMDB (optional)
Set `TMDB_API_KEY` in a `.env` file to enrich films with popularity, ratings, runtime, budget, and genre metadata. The pipeline runs without a key (genre features zero-filled with a Drama prior).

## Running the Pipeline

1. Install dependencies: `pip install numpy pandas scikit-learn xgboost requests python-dotenv`
2. Place `ml-100k/` in the project root
3. (Optional) Add `TMDB_API_KEY=<your_key>` to `.env`
4. Open and run `TheatreIQ_FirstPass.ipynb` top-to-bottom

Outputs are written to `.cache/` (TMDB enrichment) and `outputs/` (schedule artifacts).

## Revenue Impact Methodology (Section 11)

Section 11 estimates the annual ticket revenue lift if all 5 theaters adopt the Section 8 recommended schedules.

### Baseline Revenue

Revenue per showing is `seats_filled × ticket_price` as generated by `synthetic_performance.py`.  
Annual revenue is computed by multiplying average revenue per showing by assumed annual showings:

| Venue | Data Status | Assumed Annual Showings |
|---|---|---|
| Drafthouse | Well-sampled (5+ years) | Actual from data (~1,960/yr) |
| Heights Theater | Well-sampled (5+ years) | Actual from data (~1,065/yr) |
| Lagoon | Incomplete scraping | 520 projected |
| Riverview Theater | Incomplete scraping | 600 projected |
| Trylon Cinema | Incomplete scraping | 350 projected |

Assumed showings for partially-scraped venues are conservative estimates based on screen count × ~3.5 shows/screen/week. Results for those venues are labeled `* projected` in the output.

### Occupancy Lift Model

The Pearson r ≈ 0.3 correlation between match score and synthetic occupancy (Section 9) is used as the slope coefficient:

```
Δ_occupancy = r × (Δ_match_score / σ_match) × σ_occ
```

| Parameter | Value | Source |
|---|---|---|
| `r` | 0.30 | Pearson correlation, Section 9 |
| `Δ_match_score` | per-venue | mean(recommended schedule scores) − mean(historically booked scores) |
| `σ_match` | from `analysis` df | std of `max_match_score` across 491 films |
| `σ_occ` | from `analysis` df | std of `avg_occ` across 491 films |

Occupancy after lift is clipped to [0.05, 0.98].

### Adoption Scenarios

| Scenario | Assumption |
|---|---|
| **Full adoption** | All slots replaced by recommended films; lift applied to 100% of annual showings |
| **Conservative** | Lift scaled by the fraction of current slots labeled `consider_dropping` (<50 match score) — only those slots are being replaced |

### Actual Results (from notebook run)

| Venue | Baseline Annual | Full Adoption | Δ | % Lift |
|---|---|---|---|---|
| Heights Theater | $2,046,550 | $2,284,942 | +$238,392 | +11.6% |
| Drafthouse | $1,195,666 | $1,276,836 | +$81,170 | +6.8% |
| Lagoon* | $847,546 | $905,901 | +$58,355 | +6.9% |
| Riverview Theater* | $540,657 | $603,538 | +$62,881 | +11.6% |
| Trylon Cinema* | $266,637 | $290,378 | +$23,741 | +8.9% |
| **Total** | **$4.90M** | **$5.36M** | **+$0.46M** | **+9.5%** |

Conservative scenario (replace only `consider_dropping` slots): **$4.90M → $5.11M (+$0.21M, +4.3%)**

### Key Assumptions

1. All occupancy and revenue figures are **synthetic** — no real box-office data was used.
2. The model applies r = 0.3 as a moderate projection of real-world causal effect. The **actual measured Pearson r in this dataset is 0.084** (Section 9) — using 0.3 represents an assumed stronger causal relationship than the synthetic data alone shows.
3. Ticket prices held constant at **$13 evening / $10 matinee**.
4. Recommendations **substitute films** — total annual showings per venue are unchanged.
5. Venue seat capacities come from `VENUE_CAPACITY` in `synthetic_performance.py`: Trylon 175, Riverview 250, Lagoon 350, Heights 500, Drafthouse 150/screen.
6. `best_segment_score` in `venue_schedules` reflects archetype-boosted scores; baseline scores are un-boosted — Δ_score has a slight upward bias for the recommended side.
